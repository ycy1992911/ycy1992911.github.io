优先队列

发表于 2018-02-07 | 分类于 算法 第四版 | 阅读次数:

我们遇到了什么问题？

许多程序都需要处理有序的元素，但不一定要求它们全部有序，或是不一定要一次就将它们排序。很多情况下我们需要收集一些元素，处理当前最大的元素，在收集更多的元素，再处理当前最大的元素。例如，绝大多数手机分配给来电的优先级都会比游戏程序高。

又比如，我们需要从10亿个元素中选出最大的是个，这时将一个10亿规模的数组排序显然很不明智。

我们需要一种什么样的数据结构？

在以上情况下，一个合适的数据结构应该支持两种操作：删除最大元素 和 插入元素。这种数据类型叫做 优先队列 (Priority Queue)。优先队列的使用和队列（删除最老的元素）以及栈（删除最新的元素）类似，但高效地实现它则更有挑战性。

怎么实现？

对于 优先队列 来说，大致上有三种实现方法即：数组，链表 以及 二叉堆。其中基于二叉堆数据结构的实现方法，能够实现对数级别的删除最大元素和插入元素操作。

1. 数组实现（无序）

这种实现方法和栈类似。其中，要实现删除最大元素的方法，我们可以添加一段类似于选择排序的内循环代码，将最大元素和便捷元素交换然后删除它。

2. 数组实现（有序）

另一种方法就是在插入元素 add( ) 方法中添加代码，将所有较大元素向右边移动一个以使数组保持有序（和插入排序一样）。这样最大的元素总会在数组的一边，优先队列的删除最大元素操作就和栈的 pop( ) 操作一样的。

4. 二叉堆实现

二叉堆 是一组能够用堆有序的完全二叉树排序的元素，并在数组中按照层级存储（不使用数组的第一个位置）。如果用指针来表示堆有序的二叉树，那么每个元素都需要三个指针来找到它的上下节点。但如果我们使用完全二叉树，表达就会变得特别方便。完全二叉树之用数组而不需要指针就可以表示，具体方法就是将二叉树的节点按照 层级顺序 放入数组中，根节点在位置1，它的字节点在位置2和3，字节点的字节点分别在位置4和5，6和7，以此类推。

为了使堆有序，我们需要在堆内自动实现对元素的排序，此处有两种情况：

  • 由下至上的堆有序化（上浮)
    添加元素时，由于新添加的元素处于堆（内部数组）的末尾，为了维持堆有序，需要将该元素在树的结构中上浮（或者不浮动，取决于元素的大小）至合适的位置.
  • 由上至下的堆有序化（下沉)
    删除元素时，由于我们总是删除最大的元素（即树的根节点），根据数组的特性，删除末尾元素的操作时间复杂度为 O(1), 因此将根节点的元素与末尾的元素对调，并把新的末尾元素清零（或者元素值设为null，以防止对象游离）。
    此时根节点明显不是最大的元素，因此需要将根节点的值下沉到树的合适的位置。

优先队列有什么变体或改进？

1. 多叉堆

让我们先看一下三叉，即基于用数组表示的完全三叉树构造堆。对于数组中1至N的N个元素，位置k的节点大于等于位于 3k-1、3k 和 3k+1 的节点，小于等于位于 (k+1)/3 的节点。以此类推，对于给定的d，将其修改为任意的d叉树并不困难。

2. 调整数组大小

可以在 add( ) 中添加将数组长度加倍的代码，在 del( ) 中添加将数组长度减半的代码。

3. 元素的不可变性

优先队列存储了用例创建的对象，但同时假设用例代码不会改变它们（因为改变就可能打破堆的有序性）。

4. 索引优先队列

很多应用中，允许用例引用已经进入优先队列中的元素是有必要的。做到这一点的一种简单方法是给每个元素一个索引并存在一个（或多个）平行数组中。
